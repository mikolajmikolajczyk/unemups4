## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
Code-review round-7 of task-40 (GCN->SPIR-V recompiler). All 3 corpus shaders recompile to VALID portable SPIR-V passing spirv-val (verified independently) and the clamp/access-chain/madmk logic is confirmed correct — these fixes harden portability (MoltenVK/Metal north star, decision-3), the task-42 provider interface, forward-fidelity vs the interp, and a real CI flake. NONE break the current corpus. Everything is in crates/gcn (recompile.rs primarily). ITEMS: #1 MoltenVK PS inputs (recompile.rs ensure_ps_input ~1258): PS interpolants are scalar f32 Input variables decorated Location=attr + Component=chan. Valid SPIR-V, but MoltenVK/Metal may mistranslate scalar+Component (read component 0 for all channels) -> interp_color_ps wrong colors on Metal. Change to a vec4 Input per attribute Location + OpCompositeExtract the channel (the MoltenVK-reliable pattern). #6-CI-FLAKE (tests/recompile.rs): the spirv-val + golden-disasm tests write to a SHARED fixed temp path (temp_dir()/unemups4_recompile_dis.spv and similar) -> under parallel runners (cargo nextest / sharding) iterations race and corrupt each other's .spv -> spurious golden failures. Use unique temp paths (include shader name + a tempfile/pid) or an in-memory disasm. #3 coalesce IoVar (recompile.rs io_inputs ~1274): pushes a duplicate IoVar{location:attr, components:1} for EACH (attr,chan) pair, so a multi-channel attribute (interp_color_ps reads attr0 ch0/1/2) emits three IoVar{location:0} entries -> task-42's VS<->PS wiring breaks. Coalesce into one IoVar{location, components:N} per Location. #4 V# stride from descriptor (recompile.rs ensure_vs_buffer ~1333): ArrayStride + BufferBinding::stride_bytes are HARDCODED 16, ignoring the real V# stride (interp reads word1[29:16] via decode_v_sharp). A stride!=16 shader silently corrupts the fetch AND diverges from the interp. Either plumb the real stride from the SMRD-loaded V# through to ensure_vs_buffer, OR return a structured RecompileError for stride!=16 (do NOT silently assume 16). Add a LIMITATION comment + extension path at the SSBO-construction site. #5 num_records push-const layout (recompile.rs + IoLayout): num_records is emitted as a PushConstant but IoLayout carries no push_constant_size_bytes / layout descriptor, so task-42 must know by convention '4-byte block at offset 0'; this breaks when a 2nd push-const (base vertex) is added (task-53). Add an explicit push-constant layout to IoLayout (offset/size per field) so task-42 wires it correctly; document that task-42 MUST push num_records (a zero-init/missing push-const degenerates the clamp to index 0 = every fetch element 0, invisible to spirv-val + the interp). #2 portability target (tests/recompile.rs): spirv-val runs --target-env vulkan1.1, but MoltenVK = vulkan1.0 + VK_KHR_portability_subset and StorageBuffer needs SPIR-V 1.3/Vulkan1.1. Add a portability-subset validation (spirv-val --target-env vulkan1.1 is ok if we DECIDE Vulkan1.1-min; document that decision, OR validate the portability profile). Decide+document StorageBuffer (SPIR-V1.3) vs Uniform+BufferBlock (SPIR-V1.0) for the min-MoltenVK target. #6-idxen (recompile.rs emit_mubuf ~981): MUBUF idxen only recognizes vaddr==v0 as gl_VertexIndex; a VS that relocates the index into another VGPR (v_add then idxen on vN) reads an uninitialized slot (0). Track the vertex index through moves OR return a structured error if idxen vaddr is not the index-carrying reg. #F2 offen error variant: emit_mubuf rejects offen with RecompileError::Shape (conflates malformed-shader with valid-but-unsupported) — use a distinct 'deferred/unsupported' variant so task-41's harness can categorize it. #F4 VINTRP handshake doc: the DIVERGENCE comment says 'the diff harness supplies the same plane value on both sides' but doesn't name WHO/WHERE — strengthen it to state task-41 must drive the recompiled PS Location input with the oracle's computed P0+I*(P1-P0)+J*(P2-P0). LOW hygiene: decode_v_sharp doc claims all-four-words bounds-checked but reads only 3 (interp.rs — fix comment); the dead m0 OpLoad (uninitialized Function var) — drop it or give it an initializer; offen partial-state pre-loop guard missed in interp.rs (vaddr+1 bounds); portable-caps test hand-scans SPIR-V words (bypassable) — parse via the spirv crate; module doc 'phase 4'/'a later layer' temporal wording — name the concrete abstraction. Keep ps4-gcn Vulkan-free (rspirv only), no task-NN refs in new comments (conventions.md:20). Preserve the exercised corpus recompile output + spirv-val pass.
